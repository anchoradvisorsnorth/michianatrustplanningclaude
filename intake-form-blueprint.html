Tally Build Tips

  • Use Tally's multi-page form feature — each Screen above becomes one page. This prevents overwhelm and increases completion rates.
  • Set Q5 (married?) as the main branching question. All of Screen 2 should be hidden behind "Show if Q5 = Yes".
  • Q14 (minor children?) gates Q15 and Q16. Use Tally conditional logic: "Show if Q14 = Yes".
  • Q21 (real estate?) gates the entire Screen 6. If No, skip all 6 deed questions — Docupilot will omit the Quitclaim Deed from the bundle.
  • In Docupilot, create one template per document. Map each Tally field name exactly to the merge tag shown above. Then create a bundle that generates all 7 at once.
  • Q25 (legal description) will be the hardest for users. Add a help text explaining exactly where to find it — and consider a "I don't have this yet" option that flags the submission for manual follow-up.
  • The {{TRUST_DATE}} merge field (needed in Will, CoT, Deed) should be auto-populated by Docupilot as today's date — don't ask users for this.
